Why GTray exists

Built around three simple principles

  1. I don't want to check my inbox from a browser tab.
  2. Gmail's web experience is just fine — no need to reinvent it.
  3. I want to know which account needs my attention at a glance, straight from the Dock.

A browser dedicated to your mail, not another client to learn

The real Gmail

No reimplemented email here: it's Gmail's web interface, with the shortcuts, chat, and everything you already use. Links open in your browser and Compose gets its own window.

Each account, its own session

Every inbox lives in an isolated profile: sign in once and it sticks. Switch accounts instantly with ⌘1…⌘9, without logging out or mixing cookies.

The number that matters

The Dock badge adds up the unread mail of every inbox and updates the moment you read or archive — even with the window closed.

An app that touches your Gmail deserves code you can read

That's why GTray is open source and only downloadable from GitHub Releases: every version comes from code anyone can audit, built by GitHub Actions in plain sight.

MatiasSanchezCabrera/gtray
  • Your sessions never leave your Mac. They are stored locally, like in any browser. GTray has no servers and no telemetry.
  • Google never shares your password with GTray. The login happens on Google's real page; the app asks for no OAuth and no API permissions.
  • Counters are read from Gmail's own feed with your cookies, straight from Google to your Mac. Every request the app makes goes to Google — except a daily update check against gtray.app (a plain GET, no identifiers), which you can turn off.

Shortcuts

⌘1…⌘9 switch inbox ⌘N add account ⌘R reload ⌘Q quit (closing the window keeps it counting in the background)
